# バックエンド仕様整合対応

## 実施内容
- API 共通レスポンス形式を `data` / `meta` / `errors` に統一
- `requestId` を付与するフィルターと例外ハンドリングを追加
- 未実装だったサイト情報 API、作品一覧 API、コール種別マスタ API を追加
- 楽曲一覧 API のレスポンスを仕様書準拠の `items` + `pagination` 形式へ変更
- メンバー一覧 / 詳細 API を仕様書に合わせて現在所属・所属履歴・プロフィール項目まで拡張

## 補足
- 既存の `songs` / `members` 系統合テストはレスポンス構造変更に合わせて更新が必要
- `status=PUBLISHED` のサイトのみ返す前提は現状維持
