# 2026-03-23: ドメインモデル再構築 (Step 1再実行)

## 作業概要
仕様書およびER図の大幅なアップデートに伴い、Step 1で作成したドメインクラス（Entity/Enum/Repository）の再構築を実施しました。

## 主な変更点
1. **サイト拡張性の強化**
   - `Site` エンティティに `ownerUserId`, `visibility`, `isSeededSite` などを追加。
   - テーマ設定を `SiteThemeSetting` にファイル名および構造とも変更・拡張。
2. **中間テーブルの導入と履歴化**
   - メンバーの兼任や卒業などに耐えられるよう `MemberGroupHistory` を導入し、`Member` 側から `group` への直接参照を廃止。
   - 楽曲の各作品収録の要件に合わせ、`SingleRelease` と `Song` を直接結びつけるのではなく、`ReleaseSong` 中間テーブルを作成し多対多関係を表現。
3. **別名テーブルの分離**
   - 将来の検索要件拡張に備え、本体の `Song`, `Member` テーブルから検索用の別名を分離し `SongAlias`, `MemberAlias` テーブルを作成。
4. **コールマスタの導入**
   - コールの種別について、Enumではなくデータベースのレコードとして拡張管理できるように `CallTypeMaster` を追加。それに伴い `SongCallItem` は `CallType` Enum の代わりに `call_type_code` を外部キー相当の文字列として持つよう修正（旧 `CallType` Enumは非推奨化）。

## 対象ソースパッケージ（jp.co.idolFunSite.domain 配下）
- `site` (Site, SiteThemeSetting)
- `common` (Status, Visibility, BaseEntity)
- `group` (IdolGroup)
- `member` (Member, MemberGroupHistory, MemberRoleType)
- `release` (SingleRelease, ReleaseSong, ReleaseType)
- `song` (Song, SongMember, SongRoleType)
- `alias` (SongAlias, MemberAlias, AliasType)
- `call` (SongCallBlock, SongCallLine, SongCallItem)
- `master` (CallTypeMaster)

## 特記事項
ドキュメントに基づき「Spring Boot 側のEntityやRepository層の土台作成」としてのStep 1を完了しました。不要となった旧 `ThemeSetting` や `CallType` はコンパイルエラーを防ぐため内容を空（Deprecated記載のみ）として上書きしています。
